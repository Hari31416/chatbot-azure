## 1. What This Project Does

A **serverless-ready chatbot API** that:

- Accepts text messages and replies using an LLM (via LiteLLM).
- Accepts image uploads with optional text prompts and replies with vision-capable models.
- Persists conversation history in DynamoDB (with automatic TTL expiry).
- Stores uploaded images privately in S3 and returns secure, temporary presigned URLs.
- Reads secrets securely from AWS Systems Manager (SSM) Parameter Store.
- Runs locally using `uvicorn`, and in production on **AWS Lambda + API Gateway** using the **Mangum** ASGI adapter.

## 2. Architecture Overview

```
Client
  │
  ├── POST /chat              (JSON body: message, conversation_id, user_id)
  └── POST /chat/image        (multipart: file + optional message/conversation_id/user_id)
         │
         ▼
API Gateway (HTTP API v2)
         │
         ▼  ASGI event
AWS Lambda (FastAPI + Mangum adapter)
         │
         ├── DynamoDB         → Store/retrieve conversation + context
         ├── S3               → Store uploaded images
         ├── SSM Param Store  → Decrypt LITELLM_API_KEY at cold start
         └── LiteLLM          → Route completion to NVIDIA / OpenAI / etc.
```

## 4. How Each Component Works

### `main.py` — Entry Point

```python
from mangum import Mangum
app = FastAPI(title="Chatbot API")
handler = Mangum(app)   # This is what Lambda calls
```

The `handler` variable is the Lambda function's entry point. AWS Lambda calls `handler(event, context)` for every HTTP request. Mangum translates the raw Lambda event into a standard ASGI/HTTP format that FastAPI understands.

---

### `settings.py` — Configuration

Uses `pydantic-settings` to load all configuration from environment variables (or `.env` locally). Every setting has a default and a `validation_alias` matching the env var name:

```python
class Settings(BaseSettings):
    aws_region: str = Field(default="us-east-1", validation_alias="AWS_REGION")
    dynamodb_table_name: str = Field(...)
    s3_bucket_name: str = Field(...)
    litellm_model: str = Field(...)
    litellm_api_key: str | None = Field(...)
    litellm_base_url: str | None = Field(...)
    context_ttl_seconds: int = Field(default=3600)
    allowed_image_mime_types: list[str] | str = Field(...)  # comma-sep string in Lambda
```

`@lru_cache` ensures settings are loaded **once per Lambda warm instance**.

---

### `dependencies.py` — Dependency Injection

Provides singletons injected into route handlers:

- `get_settings()` — cached Pydantic settings
- `get_dynamodb_table()` — cached boto3 DynamoDB table resource
- `get_s3_client()` — cached boto3 S3 client (supports path-style for local Minio)
- `get_repository()` — creates `ConversationRepository` from table
- `get_storage()` — creates `StorageService` from S3 client
- `get_ssm_parameter()` — cached SSM SecureString lookup
- `get_llm_client()` — builds `LlmClient`, resolving API key from SSM if configured

---

### `conversation_repository.py` — DynamoDB Schema

Uses a **single-table design** with a composite key:

| Item Type                  | `pk`        | `sk`                           |
| -------------------------- | ----------- | ------------------------------ |
| Conversation metadata      | `CONV#<id>` | `META`                         |
| User/assistant message     | `CONV#<id>` | `MSG#<timestamp>#<message_id>` |
| Conversation context (TTL) | `CONV#<id>` | `CTX`                          |

The `CTX` item stores the last N messages as a JSON list and has a `ttl` attribute (Unix epoch) for automatic DynamoDB TTL expiry — it disappears after `CONTEXT_TTL_SECONDS` (default 1 hour).

---

### `storage.py` — S3 Image Storage

- `upload_image(key, data, mime_type)` — uploads raw bytes to S3, returns an `UploadResult` dataclass.
- `generate_presigned_url(key, expiration_seconds=3600)` — generates a temporary signed GET URL so a browser can display a private S3 object without making the bucket public.

```python
def generate_presigned_url(self, key: str, expiration_seconds: int = 3600) -> str:
    return self._s3.generate_presigned_url(
        "get_object",
        Params={"Bucket": self._bucket, "Key": key},
        ExpiresIn=expiration_seconds,
    )
```

Image keys are stored as `<conversation_id>/<message_id>.<ext>`.

---

### `llm.py` — LiteLLM Wrapper

A thin async wrapper around `litellm.acompletion`:

```python
async def generate(self, messages: list[dict]) -> str:
    response = await acompletion(
        model=self._model,
        messages=messages,
        api_key=self._api_key,
        base_url=self._base_url,
    )
    return response.choices[0].message.content
```

Supports any provider LiteLLM supports (OpenAI, Anthropic, NVIDIA NIM, etc.) by changing the model string and base URL.

---

### `routes.py` — API Endpoints

Both endpoints are wrapped in `try...except` so that any error (LLM API failure, DynamoDB timeout, etc.) is caught and returned as a clean JSON response with an `error` field — instead of crashing with a generic 500 Internal Server Error.

```python
@router.post("/chat", response_model=ChatResponse)
async def chat(payload: ChatRequest, ...) -> ChatResponse:
    try:
        # ... normal logic
        return ChatResponse(conversation_id=..., assistant_message=..., ...)
    except Exception as e:
        return ChatResponse(error=str(e))
```

## 6. All Changes Made & Why

### Bug Fix A — FastAPI 422 Unprocessable Entity on `/chat`

**Problem**: The dependencies `get_repository`, `get_storage`, `get_llm_client` in `dependencies.py` had an optional parameter `settings: Settings | None = None`. Because `Settings` is a Pydantic model, FastAPI mistakenly tried to parse it from the request body, causing 422 errors on every `/chat` request.

**Fix**: Removed the unused `settings` parameter from all three dependency functions. They now call `get_settings()` directly.

```python
# Before (broken)
def get_repository(settings: Settings | None = None) -> ConversationRepository:
    _ = settings or get_settings()

# After (fixed)
def get_repository() -> ConversationRepository:
    table = get_dynamodb_table()
    return ConversationRepository(table)
```

---

### Bug Fix B — Pydantic Settings Crash on `ALLOWED_IMAGE_MIME_TYPES`

**Problem**: Lambda passes list-type env vars as comma-separated strings (e.g., `image/png,image/jpeg`). Pydantic's `BaseSettings` tried to JSON-decode the string before the custom validator could run, throwing a crash at every cold start.

**Fix**: Changed the type hint from `list[str]` to `list[str] | str`. Pydantic now accepts the raw string and the `@field_validator` splits it by comma.

```python
# Before (crash in Lambda)
allowed_image_mime_types: list[str] = Field(...)

# After (works in Lambda)
allowed_image_mime_types: list[str] | str = Field(...)
```

---

### Change C — Added Mangum ASGI Handler

**Why**: AWS Lambda calls a handler function, not a web server. The Mangum library translates Lambda's event format into ASGI format so FastAPI can respond normally.

**What was added** to `main.py`:

```python
from mangum import Mangum
handler = Mangum(app)
```

The SAM template's `Handler: app.main.handler` points Lambda to this variable.

---

### Change D — S3 Presigned URL for Private Image Access

**Why**: The S3 bucket is fully private (no public access). Returning a raw S3 key to a frontend would be useless — the browser can't access a private bucket. Presigned URLs solve this by generating a temporary signed GET URL (valid 1 hour) that any browser can use without credentials.

**What was added** to `storage.py`:

```python
def generate_presigned_url(self, key: str, expiration_seconds: int = 3600) -> str:
    try:
        return self._s3.generate_presigned_url(
            "get_object",
            Params={"Bucket": self._bucket, "Key": key},
            ExpiresIn=expiration_seconds,
        )
    except Exception:
        return f"https://{self._bucket}.s3.amazonaws.com/{key}"
```

The `/chat/image` endpoint now calls this and includes the URL in the `attachment.presigned_url` field of the response.

---

### Change E — Secure SSM Parameter Lookup for LiteLLM API Key

**Why**: Storing API keys in plain-text Lambda environment variables is bad practice — they're visible to anyone with AWS console access. SSM Parameter Store with `SecureString` type encrypts keys using KMS and only decrypts them at runtime, scoped by IAM role.

**What was added** to `dependencies.py`:

```python
@lru_cache
def get_ssm_parameter(param_name: str) -> str | None:
    try:
        ssm = boto3.client("ssm", region_name=get_settings().aws_region)
        response = ssm.get_parameter(Name=param_name, WithDecryption=True)
        return response["Parameter"]["Value"]
    except Exception:
        return None

def get_llm_client() -> LlmClient:
    settings = get_settings()
    api_key = settings.litellm_api_key  # fallback for local dev

    ssm_param_name = os.getenv("LITELLM_API_KEY_PARAMETER")
    if ssm_param_name:
        ssm_key = get_ssm_parameter(ssm_param_name)
        if ssm_key:
            api_key = ssm_key  # override with SSM value in production

    return LlmClient(model=..., api_key=api_key, base_url=...)
```

In production, Lambda has the `LITELLM_API_KEY_PARAMETER=/chatbot/litellm_api_key` env var set. The app fetches and decrypts the key from SSM. Locally, it falls back to whatever is in `.env`.

---

### Change F — Added `LITELLM_BASE_URL` Parameter to SAM Template

**Why**: LiteLLM needs a custom `base_url` to call NVIDIA's NIM endpoints instead of OpenAI's default. The value `https://integrate.api.nvidia.com/v1` was hardcoded as a SAM parameter default.

**What was added** to `template.yaml`:

```yaml
Parameters:
  LiteLlmBaseUrl:
    Type: String
    Default: https://integrate.api.nvidia.com/v1

Globals:
  Function:
    Environment:
      Variables:
        LITELLM_BASE_URL: !Ref LiteLlmBaseUrl
```

---

### Change G — Graceful Error Handling in All Endpoints

**Why**: Any unhandled exception in FastAPI becomes a generic `500 Internal Server Error` with no useful detail. This is especially painful when debugging LLM API failures or DynamoDB issues in production. We now catch all exceptions and return them as part of the JSON response body.

**What changed** in `routes.py` — both `/chat` and `/chat/image` now have:

```python
try:
    # ... all normal logic
    return ChatResponse(...)
except Exception as e:
    error_msg = e.detail if isinstance(e, HTTPException) else str(e)
    return ChatResponse(error=error_msg)
```

**What changed** in `schemas.py` — all fields made optional with `None` defaults, and `error` field added:

```python
class ChatResponse(BaseModel):
    conversation_id: str | None = None
    assistant_message: str | None = None
    error: str | None = None  # populated on failure, null on success
    # ...
```

### SAM Template Fixes During Deployment

| Error                          | Root Cause                                                                                                                                                      | Fix                                                                                                                              |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `BinaryMediaTypes not defined` | `AWS::Serverless::HttpApi` (HTTP API v2) does not support this property — only REST API v1 does. HTTP API handles binary payloads natively.                     | Removed the `BinaryMediaTypes` block entirely.                                                                                   |
| `Reserved key AWS_REGION`      | Lambda automatically sets `AWS_REGION` as a built-in runtime variable. Re-declaring it in `Environment.Variables` causes a 400 error from Lambda's API.         | Removed `AWS_REGION: !Ref AWS::Region` from the template — boto3 picks it up from the built-in runtime variable automatically.   |
| `ROLLBACK_COMPLETE state`      | When a stack fails its first-ever creation, CloudFormation rolls it back but leaves a dead shell. This shell cannot be updated — only deleted and recreated.    | `aws cloudformation delete-stack --stack-name <name>` then `sam deploy` again.                                                   |
| `S3 Bucket does not exist`     | The SAM-managed CloudFormation stack (`aws-sam-cli-managed-default`) was deleted, but `samconfig.toml` still referenced the old managed S3 bucket.              | `aws cloudformation delete-stack --stack-name aws-sam-cli-managed-default` then `sam deploy --guided` to recreate.               |
| `pip editable install error`   | `uv export` by default includes the local project as an editable install (`file:///tmp/...`), which pip inside the SAM Docker container cannot hash or resolve. | `uv export --format requirements-txt --no-hashes --no-emit-project -o requirements.txt`                                          |
| Cross-architecture warning     | Building an `x86_64` Lambda image on an Apple Silicon (`arm64`) host causes Docker to emulate x86, which is slow and fragile.                                   | Added `Architectures: [arm64]` to `Globals.Function` in `template.yaml`. Lambda Graviton (arm64) is also 20% cheaper and faster. |

## 7. AWS SAM Deployment Guide

### Prerequisites

```bash
# Install AWS SAM CLI
brew install aws-sam-cli

# Authenticate AWS CLI
aws login
aws configure   # Set region: ap-south-1
```

### First Time Setup

```bash
# Store your LLM API key in SSM
aws ssm put-parameter \
  --name "/chatbot/litellm_api_key" \
  --type "SecureString" \
  --value "nvapi-..." \
  --overwrite

# Generate requirements.txt (must be re-run after any dependency change)
cd backend
uv export --format requirements-txt --no-hashes --no-emit-project -o requirements.txt
cd ..

# Build and deploy
sam build --use-container
sam deploy --guided   # Only needed first time, saves settings to samconfig.toml
```

### Subsequent Deployments

```bash
cd backend
uv export --format requirements-txt --no-hashes --no-emit-project -o requirements.txt
cd ..
sam build --use-container
sam deploy
```

### Verify Deployment

```bash
curl https://ltr80pcnvd.execute-api.ap-south-1.amazonaws.com/health
# Returns: {"status": "ok"}
```

```bash

  --name "/chatbot/litellm_api_key" \
```

## 9. API Endpoints

### `GET /health`

Health check. Returns `{"status": "ok"}`.

---

### `POST /chat`

Send a text message.

**Request Body (JSON)**:

```json
{
  "message": "What is the capital of France?",
  "conversation_id": "optional-uuid-to-continue-a-conversation",
  "user_id": "optional-user-identifier"
}
```

**Success Response**:

```json
{
  "conversation_id": "uuid",
  "user_message_id": "uuid",
  "assistant_message_id": "uuid",
  "assistant_message": "The capital of France is Paris.",
  "created_at": "2026-05-23T10:00:00Z",
  "error": null
}
```

**Error Response** (e.g., LLM API failure):

```json
{
  "conversation_id": "uuid-or-unknown",
  "assistant_message": null,
  "error": "APIConnectionError: ..."
}
```

---

### `POST /chat/image`

Send an image (with optional text prompt).

**Request**: `multipart/form-data`

- `file` — image file (PNG, JPEG, or WebP, max 5MB)
- `message` — optional text prompt
- `conversation_id` — optional
- `user_id` — optional

**Success Response**:

```json
{
  "conversation_id": "uuid",
  "user_message_id": "uuid",
  "assistant_message_id": "uuid",
  "assistant_message": "This image shows...",
  "created_at": "2026-05-23T10:00:00Z",
  "attachment": {
    "s3_key": "conv-uuid/msg-uuid.png",
    "mime_type": "image/png",
    "size_bytes": 45000,
    "presigned_url": "https://chatbot-uploads-....s3.amazonaws.com/...?X-Amz-Signature=..."
  },
  "error": null
}
```

## 10. Managing Your Deployment

### Delete the entire deployment (all services)

```bash
sam delete
# Prompts y/N twice (stack + S3 artifacts). Both yes.
```

This deletes: Lambda, API Gateway, DynamoDB Table (+ all data), S3 bucket (+ all images), IAM Role.  
This does NOT delete: SSM Parameter, SAM managed S3 bucket.

### Redeploy after deletion

```bash
sam build --use-container && sam deploy
```

### Tail live Lambda logs

```bash
sam logs -n ChatbotBackendFunction --stack-name chat --tail
```

### Check all CloudFormation events

```bash
aws cloudformation describe-stack-events --stack-name chat
```

### Check the Deployed URL

```bash
aws cloudformation describe-stacks \
  --stack-name chat \
  --query "Stacks[0].Outputs[?OutputKey=='ApiUrl'].OutputValue" \
  --output text
```
